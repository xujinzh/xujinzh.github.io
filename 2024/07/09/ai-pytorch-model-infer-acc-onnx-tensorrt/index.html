使用 tensorrt 加速 pytorch 模型推理

发表于2024-07-09|更新于2024-07-09|researchdeep learning
|字数总计:4.1k|阅读时长:23分钟|阅读量:

NVIDIA® TensorRT™ 是用于高性能深度学习推理的 API 生态系统。TensorRT 包括推理运行时和模型优化，可为生产应用提供低延迟和高吞吐量。TensorRT 生态系统包括 TensorRT、TensorRT-LLM、TensorRT 模型优化器和 TensorRT Cloud。本篇介绍如何使用 tensorrt 加速 pytorch 模型推理，以及常用的推理方案对比。

TensorRT 简介

NVIDIA TensorRT 模型优化器是一个统一的先进模型优化技术库，包括量化、稀疏和蒸馏。它可压缩深度学习模型，以便在 TensorRT-LLM 和 TensorRT 等下游部署框架中高效优化 NVIDIA GPU 上的推理。TensorRT 直接集成到 PyTorch、Hugging Face 和 TensorFlow 中，通过一行代码实现 6 倍的推理速度。TensorRT 提供 ONNX 解析器，可将热门框架中的 ONNX 模型导入到 TensorRT。通过 GPU Coder，MATLAB 与 TensorRT 集成，可自动为 NVIDIA Jetson™、NVIDIA DRIVE® 和数据中心平台生成高性能推理引擎。更多介绍请参考NVIDIA TensorRT

下面评测 pytorch 模型在不同推理方案下的速度。

结论

通过上面的实测，可以知道：

  1. 采用 torch_tensorrt 和 tensorrt 推理速度是最快的。
  2. 相比于原生 pytorch 推理，onnxruntime 有时推理速度更慢。

另外，torch_tensorrt 相比于 tensorrt 更加方便使用，当有些算子在 tensorrt 中不支持时，会自动调用 pytorch 默认算子运行，给开发人员使用更加灵活方便。

可能遇到的问题

  1. tensorrt 版本不匹配

如果使用 trtexec 把 onnx 转为 trt 文件，那么在使用 python 代码调用 trt 文件时会可能会出现反序列化版本不匹配问题。这是因为安装的 pip install tensorrt 的版本和手动下载的 tensorrt(https://developer.nvidia.com/downloads/compute/machine-learning/tensorrt/10.0.1/tars/TensorRT-10.0.1.6.Linux.x86_64-gnu.cuda-11.8.tar.gz) 里的 bin/trtexec 不匹配导致的。因此，最好使用上面的 python 代码转换 onnx 为 trt 的方式。或者，在 TensorRT-10.0.1.6.Linux.x86_64-gnu.cuda-11.8.tar.gz 里的 python 文件夹下安装 tensorrt。
